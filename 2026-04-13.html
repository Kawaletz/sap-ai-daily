NEU Windows-Guide: Claude AI mit SAP ABAP-System via MCP verbinden

  • Ein neuer SAP-Community-Blog-Post liefert eine vollständige Windows-Schritt-für-Schritt-Anleitung, wie man Claude Desktop direkt mit einem live SAP ABAP-System über das Model Context Protocol (MCP) verbindet – ohne VSCode oder externes Tooling.
  • Der Autor setzt zwei komplementäre MCP-Server ein: mcp-abap-adt (Read-only, 13 Tools wie GetProgram, GetClass, GetFunction, SearchObject) und mcp-abap-abap-adt-api (Vollständiges CRUD mit 30+ Tools inkl. Transport-Management und Code-Aktivierung).

NEU Joule A2A: Drei Integrationsansätze für Code-basierte Agenten

  • Ein neuer SAP-Community-Blog-Post („Integrating Joule with Code-based Agents: Three Paths Forward") analysiert systematisch die drei größten Herausforderungen bei der A2A-Integration externer Code-Agenten in Joule – und liefert für jede einen konkreten Lösungsweg.
  • Problem 1 – Timeout: Joules synchrone A2A-Kommunikation hat ein hartes 60-Sekunden-Timeout. Komplexe Reasoning-Chains oder Agenten mit langsamen externen APIs überschreiten dieses Budget regelmäßig. Lösung: Ein asynchrones Streaming-Proxy-Muster mit SSE (Server-Sent Events) als Puffer.
  • Problem 2 – Agent Builder Limitation: Direkte A2A-Integration ist nur für Pro-Code-Agenten aus dem Joule Studio Code Editor verfügbar. Low-Code/No-Code-Agenten aus dem Joule Studio Agent Builder haben bisher kein A2A-Binding – Roadmap-Item für H2 2026.
  • Problem 3 – Authentifizierung: Ohne IAS-Absicherung sind A2A-Endpunkte potenziell offen. Der Blog beschreibt eine referenzfähige Architektur mit dem A2A Client Service als Bindeglied – POST /a2a/stream_normalised als zentraler Endpunkt für die Joule-Anbindung.
  • TIPP Für Entwickler, die eigene Python- oder Node.js-Agenten in Joule integrieren wollen, ist dieser Blog aktuell der umfassendste technische Leitfaden im SAP-Ökosystem.

NEU Joule A2A: Sicherheitsabsicherung mit IAS und JWT-Token-Validierung

  • Ein weiterer neuer SAP-Blog („Joule A2A: Secure Code Based Agents with IAS") schließt die Sicherheitslücke in der A2A-Integrationskette: Wie werden Code-basierte Agenten gegen unbefugten Zugriff abgesichert, wenn sie als A2A-Endpunkte in Joule registriert sind?
  • Ansatz: OAuth2 Client Credentials Flow über SAP Identity Authentication Service (IAS) – App-zu-App-Authentifizierung ohne Benutzerinteraktion. IAS stellt ein JWT aus, das der Agent-Middleware zur Validierung übergeben wird.
  • Implementierung: Python-Middleware prüft JWT-Tokens aus IAS, validiert Scopes und hält Agent-Discovery-Endpunkte (für A2A-Protokoll-Compliance) bewusst öffentlich – alle Aktionsendpunkte hingegen sind durch Token-Validierung geschützt.
  • Der Blog enthält funktionstüchtigen Sample-Code auf GitHub – geeignet als Vorlage für produktionsreife A2A-Agenten-Deployments auf SAP BTP.

UPDATE Performance-Based Testing: 65 SAP-Zertifizierungen (61 %) nun hands-on – Rollout abgeschlossen

  • Das neue SAP-Zertifizierungsmodell mit Performance-Based Testing (PBT) ist seit April 2026 vollständig in Kraft: 65 der insgesamt 107 Zertifizierungen (61 %) wurden auf praxisbasierte, hands-on Prüfungsformate umgestellt. Alle verbleibenden Zertifizierungen sollen in H1 2026 folgen.
