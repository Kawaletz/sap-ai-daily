Custom Code Migration Agent im Detail: Wie der erste CCM-Agent ECC-Backlogs automatisiert UPDATE

  • Was der Agent konkret tut: Der Mass S/4 Custom Code Conversion Agent verbindet sich über den ABAP Language Server mit dem ECC-System, scannt die gesamte Custom-Code-Landschaft und gleicht jedes Objekt gegen die S/4HANA Simplification Database ab. Dabei werden ATC-Verstöße automatisch katalogisiert und priorisiert.
  • Zwei Fix-Modi: Der Agent kann ATC-Probleme entweder mit deterministischen Regeln (für bekannte Transformationsmuster) oder mit KI-generierten Fixes (für komplexe, kontextabhängige Fälle) lösen. Menschliche Entwickler übernehmen die Review- und Governance-Rolle, nicht die Massenverarbeitung.
  • Realistische Einsparungen: Bei ECC-Landschaften mit 30.000+ Custom-Objekten dauerte die klassische Code-Remediation bisher 12–24 Monate Vollzeit-ABAP-Entwicklung. Der Agent zielt darauf ab, diesen Prozess auf einen teilautomatisierten Workflow zu reduzieren, bei dem Entwickler nur noch Edge Cases und Reviews behandeln.
  • Verfügbarkeit und Voraussetzungen: GA seit Q2 2026; verfügbar für SAP BTP ABAP Environment, S/4HANA Cloud Public und Private Edition (ab Release 2021). Aktives Joule for Developers Abonnement erforderlich – nach September 2026 kostenpflichtig über AI Units.

SAP Cloud AI

CAP cds 10 – Major Release am 26. Juni: Node.js 22, SQLite-Default, Vitest & Fetch API NEU

  • Termin übermorgen: Am 26. Juni 2026 erscheint @sap/cds v10 – das nächste Major Release des SAP Cloud Application Programming Model (CAP). Die offiziellen Release Notes sind bereits auf cap.cloud.sap veröffentlicht.
  • Node.js 22 als Minimum: cds 10 setzt Node.js 22 als Mindestversion voraus (Node.js 24 LTS empfohlen, Node.js 26 ab Oktober 2026). Projekte, die noch Node.js 18 oder 20 verwenden, müssen vor dem Upgrade aktualisieren.
  • Nativer SQLite-Default: Der native node:sqlite-Treiber löst better-sqlite3 als Standard-SQLite-Backend ab. Wer better-sqlite3 oder sql.js weiter nutzen möchte, muss dies explizit als Abhängigkeit deklarieren.
  • Fetch API statt Axios: Sämtliche HTTP-Client-Abhängigkeiten (inkl. Axios) werden durch Nodes native Fetch API ersetzt – schlankeres Bundle, weniger Drittanbieter-Abhängigkeiten, moderner Stack.
  • Vitest-Support und Attachment-Sicherheit: cds.test unterstützt jetzt Vitest als primäres Test-Framework (kompatibel bleibt Jest, Mocha, Node Test Runner). Neu: Datei-Uploads werden automatisch auf Malware geprüft (auch beim Download), Dateinamen werden beim Upload automatisch dedupliziert und Upload-Größe sowie -Typ sind über Annotationen einschränkbar.
  • CAP Java 5 parallel: Zusammen mit cds 10 erscheint CAP Java Major Version 5 mit Spring Boot 4 Support. Migrationsguide bereits auf capire veröffentlicht.

SAP Business AI Platform: BTP-Konsolidierung – ein Stack, drei Schichten UPDATE

  • Was sich verändert: Mit Sapphire 2026 hat SAP eine strukturelle Vereinfachung eingeleitet: SAP BTP, SAP Business Data Cloud und SAP Business AI werden zu einer konsolidierten Plattform zusammengeführt. Ziel ist die Vermeidung von Enterprise-AI-Fragmentierung – Daten, Build-Tools und Governance in einem Stack statt drei getrennten Produkten.
  • Die drei Schichten im Überblick: (1) Context Layer: SAP Business Data Cloud + Knowledge Graph liefern semantisch angereicherte Geschäftsdaten als KI-Kontext. (2) Build Layer: Joule Studio 2.0 als zentrale Entwicklungsumgebung für Custom Agents. (3) Governance Layer: AI Agent Hub – Tracking, Management und Monitoring aller Agenten unternehmensweit.
  • Konsequenz für ABAP-Entwickler: ABAP-Systeme sind in diesem Modell als „Capability Provider“ im Process Layer positioniert. Clean Core ist keine optionale Empfehlung mehr, sondern technische Voraussetzung für die Integration als Agenten-Backend.
  • NVIDIA-Integration: Joule Studio 2.0 nutzt NVIDIA OpenShell als sichere Runtime für rechenintensive Agenten – relevant für multimodale und Echtzeit-Anwendungsfälle in der Kombination mit SAP-Geschäftsdaten.
