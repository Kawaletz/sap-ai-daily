Kritische Analyse: „SAP hat zwei MCPs – eines kann man nicht nutzen, eines sollte man nicht“ TIPP

  • Mario de Felipe auf Medium (Mai 2026): SAP setzt MCP an zwei sehr unterschiedlichen Stellen ein. MCP 1: Der Integration Suite MCP Gateway (GA Q2 2026) exponiert ausgewählte APIs als gehostete MCP-Server – im Grunde ein weiteres Protokoll neben REST und OData. MCP 2: SAP nutzt intern MCP mit Joule, um auf den SAP Knowledge Graph, Ontologien und Business-Semantiken zuzugreifen – dieser interne MCP ist für externe Entwickler nicht erreichbar.
  • Kernkritik: SAP sollte semantische Modelle, Datenprodukte und analytische Modelle über die MCP-Architektur freigeben, statt nur technische API-Endpunkte. Die „Harness Engineering“-Chance werde damit verpasst. Pflichtlektüre für alle, die MCP-basierte Agenten für SAP bauen – erklärt, warum der Integration Suite MCP Gateway strukturell begrenzt ist.
