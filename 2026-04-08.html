ABAP Classic AI

NEU abap-ai/mcp – Nativer ABAP MCP Server SDK

  • Ein neues Open-Source-Projekt auf GitHub implementiert das Model Context Protocol (MCP) direkt in ABAP – serverseitig, vollständig ohne externe Node.js- oder Python-Abhängigkeiten.
  • Das SDK unterstützt alle drei MCP-Kern-Primitiven gemäß Spezifikation 2025-03-26: Prompts, Resources und Tools – damit lassen sich vollwertige MCP-Server nativ in ABAP bauen.
  • Installation erfolgt über abapGit, das SDK ist provider-agnostisch und kann mit OpenAI, Claude, Gemini und anderen LLMs zusammenarbeiten.
  • Besonderheit: Teile des SDKs wurden selbst mit KI generiert (hauptsächlich Claude Sonnet 3.7/4 und GitHub Copilot) – ein Zeichen für die Reife der ABAP-AI-Toolchain.
  • Aktuell noch ohne SSE-Streaming-Support (Echtzeit-Benachrichtigungen nicht verfügbar), ein Discord-Kanal für Community-Fragen ist eingerichtet.
  • 🔗 GitHub: abap-ai/mcp – ABAP MCP Server SDK

NEU akivaMishan/sap-mcp-server – Eclipse ADT Bridge Ansatz

  • Ein weiterer neuer MCP-Server-Ansatz: akivaMishan/sap-mcp-server setzt auf ein Eclipse ADT Bridge Plugin als Proxy – der MCP-Server kommuniziert über das Eclipse-Plugin direkt mit dem SAP ABAP-System via ADT REST API.
  • Funktionen: Suche nach ABAP-Objekten (Klassen, Programme, Pakete, Tabellen), Quellcode lesen, Paketinhalte browsen, Objekt-Metadaten abrufen, Systemkonnektivität prüfen.
  • Besonderheit: Verwendet Browser-basierte Session-Cookie-Authentifizierung (kein direkter Username/Passwort-Flow) – Authentifizierung muss periodisch erneuert werden.
  • Voraussetzungen: Node.js 18+, SAP BTP ABAP Environment, Chrome-Browser. Zielgruppe sind primär Entwickler auf SAP BTP ABAP Cloud.
  • Dieser Ansatz ergänzt die wachsende Landschaft von ABAP MCP-Servern um eine weitere Architekturvariante neben den bekannten Node.js-basierten Lösungen.
  • 🔗 GitHub: akivaMishan/sap-mcp-server

SAP Cloud AI

NEU SAP HANA Cloud wird agentisch: Discovery Agent & Data Agent

  • SAP startet die Beta-Registrierung für Data Agent und Discovery Agent in SAP HANA Cloud – ein Paradigmenwechsel hin zu agentischen Datenbankinteraktionen.
  • Der Discovery Agent fungiert als intelligenter Daten-Kompass: Er identifiziert relevante Tabellen, Views, Spalten und User-Defined Functions für einen beliebigen Business-Kontext über einen Knowledge Graph.
  • Der Data Agent übersetzt natürlichsprachliche Anfragen direkt in ausführbare SQL-Queries – keine tiefen SQL-Kenntnisse mehr notwendig für komplexe Datenmodelle.
  • Technisches Fundament: Der Custom Database Objects Knowledge Graph (geplant QRC1 2026) wird automatisch aus dem System-Katalog generiert und erfasst Tabellen, Beziehungen, Annotationen und mehr.
  • Business-Impact: Konvergenz von KI-gesteuerter Query-Generierung und Knowledge Graph öffnet SAP HANA Cloud für Business-Anwender ohne SQL-Expertenwissen.

NEU Custom Joule Agent für SAP Ariba SLP – POC in 7 Tagen

  • Ein neuer SAP-Community-Blog-Post zeigt eindrucksvoll, wie ein Custom Joule Agent für die Lieferantenregistrierungs-Genehmigung in SAP Ariba Supplier Lifecycle & Performance (SLP) in weniger als 7 Tagen als Proof-of-Concept realisiert wurde.
  • Typischer Prozess ohne KI: Lieferant füllt Fragebogen aus → Genehmigungsaufgabe landet in einer Queue → manuelles Review, Prüfung von Dokumenten und Zahlen → Entscheidung (Genehmigen / Zurücksenden / Ablehnen).
  • Mit dem Custom Joule Agent wird dieser Prozess automatisiert: Der Agent analysiert Fragebögen, prüft Compliance-relevante Daten und gibt fundierte Empfehlungen – ohne manuellen Eingriff für Standardfälle.
  • Der Entwickler hatte zuvor dieselbe Automatisierung mit SAP Build Process Automation umgesetzt – der Joule-Ansatz war deutlich schneller und direkter zu implementieren.
  • Zeigt das Potenzial von Joule Studio für Procurement-Prozesse und die Ausweitung agentischer AI in SAP LOB-Anwendungen über ABAP hinaus.
