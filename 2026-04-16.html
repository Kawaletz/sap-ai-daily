ABAP Classic AI

UPDATE SAP Cloud SDK for JavaScript v4.6.0: Connectivity-Fixes mit Auswirkung auf ABAP-/RFC-Schnittstellen in AI-Agent-Pipelines

  • Am 14. April 2026 hat das SAPExpert.AI-Team in einer gesonderten News-Analyse auf das Release SAP Cloud SDK for JavaScript v4.6.0 hingewiesen – mit Connectivity-Fixes, die für ABAP-Entwickler unmittelbar relevant sind. Hintergrund: Viele AI-Agenten, die gegen On-Premise-ABAP fahren, nutzen Node.js-Destinations über den Cloud SDK als JWT- und mTLS-Brücke.
  • Konkrete Auswirkungen: Die Fixes betreffen u. a. Principal-Propagation-Header bei Cloud-Connector-Destinations und die Fehlerbehandlung bei Retry-Zyklen gegen RFC-Destinations. Wer LangChain-/CrewAI-Agenten mit dem Cloud SDK verdrahtet, sollte Destinations-Tests vor dem nächsten Deployment wiederholen.
  • TIPP Wichtig für Teams, die AI-Agenten mit SAP Joule A2A koppeln: Die neue Version behebt subtile Header-Weitergabe-Bugs, die in Multi-Agent-Ketten zu verpassten Zertifikats-Hops führen konnten. Upgrade lohnt sich v. a. bei Cloud-Foundry-Deployments mit SCC-Backend-Anbindung.

NEU SAP Cloud Infrastructure: Deutsche Rechenzentren erhalten IT-Grundschutz-Zertifizierung – Zero-Downtime-Migration für bestehende Tenants

  • Heute, 16. April 2026, meldet das SAP News Center: Die deutschen Rechenzentren der SAP Cloud Infrastructure haben die IT-Grundschutz-Zertifizierung nach BSI-Standard erhalten. Damit erfüllt SAP jetzt einen der schärfsten deutschen Compliance-Benchmarks – relevant v. a. für Finanzsektor, öffentliche Verwaltung und kritische Infrastrukturen.
  • Technischer Kern für AI-Architekten: Bestehende Tenants profitieren von einer Zero-Downtime-Migration auf die zertifizierte Plattform – kein Re-Onboarding, keine API-Änderung. Für Kunden, die Generative AI Hub- und Joule-Workloads auf BTP Germany betreiben, bedeutet das eine Compliance-Aufwertung ohne Architektur-Refactor.
  • Strategisch: Die Zertifizierung flankiert das Delos-Cloud-Programm (GA bis Ende 2026) und positioniert SAP als souveräne Alternative zu Hyperscaler-Regionen für BAFin- und Bundesbehörden-regulierte Gen-AI-Projekte.
  • TIPP Wer eine Joule-/CAP-AI-Landschaft für deutsche Behörden oder Banken positioniert, kann ab sofort die IT-Grundschutz-Konformität als Argument in Ausschreibungen ins Feld führen – bislang war das eine klassische Lücke der SAP-BTP gegenüber deutschen Hyperscaler-Regionen.

NEU SAP Community: „Connecting custom Joule Agents to MCP Servers" – POC-Architektur für Custom-Tool-Provider

  • Ein neuer Community-Blogpost zeigt eine POC-Referenzarchitektur, wie Custom Joule Agents (aus Joule Studio) an einen eigenen MCP-Server angebunden werden, der unabhängig versioniert und auf Cloud Foundry deployed wird.
  • Technische Schlüsselpunkte: Destination im Joule-Studio-Subaccount zeigt auf die MCP-Server-URL, der Server wird im Agent-Config als Tool Provider registriert – damit stehen die MCP-Tools innerhalb jeder Joule-Agent-Skill dynamisch zur Verfügung. Im Gegensatz zur rein codebasierten A2A-Kopplung funktioniert das auch mit dem Low-Code Agent Builder.
  • TIPP Wichtige Abgrenzung für Architekten: Der Beitrag schließt die Lücke zur Agent-Builder-Low-Code-Welt, die laut SAP-Roadmap bislang keine native A2A-Integration hat – MCP wird damit zum pragmatischen Brückenschlag zwischen No-Code-Joule-Agenten und bestehenden Code-Tools.
